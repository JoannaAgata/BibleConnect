# Entwicklungsplan

## Ziel

Die Entwicklung von BibleConnect erfolgt schrittweise. Jede Entwicklungsphase baut auf der vorherigen auf und wird separat dokumentiert. 
Dadurch bleibt der Entwicklungsprozess nachvollziehbar und die einzelnen Komponenten können unabhängig voneinander getestet werden.

## Entwicklungsschritte

| Schritt | Beschreibung |
|----------|--------------|
| Schritt 1 | Projektstruktur erstellen |
| Schritt 2 | Frontend entwickeln |
| Schritt 3 | API Gateway entwickeln |
| Schritt 4 | User Service entwickeln |
| Schritt 5 | Bible Service entwickeln |
| Schritt 6 | Comment Service entwickeln |
| Schritt 7 | Group Service entwickeln |
| Schritt 8 | Notification Service entwickeln |
| Schritt 9 | Integration und Tests |
